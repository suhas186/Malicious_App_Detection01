Malicious App Detection

An Android-focused project that flags risky apps by analyzing permissions, manifest metadata, static features (e.g., API calls), and optional network/runtime signals. This page is meant to be added as index.html in your GitHub repository or published via GitHub Pages.

Overview

Goal: Help users and analysts identify potentially malicious Android apps using transparent, explainable features (permissions, components, intent filters, strings) and a machine‚Äëlearning model (e.g., SVM/RandomForest or on‚Äëdevice TensorFlow Lite). Inspired by classic research like DREBIN and modern Android privacy changes.

  • Package visibility (Android 11+): Your scanner should respect Android's package visibility rules; only query visible apps or justify QUERY_ALL_PACKAGES when absolutely necessary.
  • Permissions: Focus on runtime ("dangerous") permissions alongside other features; analyze combinations rather than single permissions in isolation.

Setup

Local training (Python)

# 1) Create env and install
python -m venv .venv && source .venv/bin/activate  # Windows: .venv\\Scripts\\activate
pip install -r requirements.txt  # scikit-learn, pandas, joblib, tflite-runtime(optional)

# 2) Extract features from APKs (outputs CSV/Parquet)
python -m features.extract --apks ./data/apks --out ./data/features.parquet

# 3) Train a model
python -m src.train --features ./data/features.parquet --model ./models/rf.joblib

# 4) Score new APK
python -m src.score --apk path/to/app.apk --model ./models/rf.joblib --report ./reports/app.md

Android 11+

Package Visibility

Query only what you need. If you must scan installed apps, declare proper visibility and justify QUERY_ALL_PACKAGES per policy. Prefer filtering by launcher‚Äëvisible apps.

Permissions

Runtime ‚ÄúDangerous‚Äù

Interpret permission combinations in context (SMS, Contacts, Storage, Accessibility). Do not infer malice from a single permission alone.

Safety

Handling Malware

Do not push malicious APKs to public repos. Keep samples offline, store hashes/features only, and use isolated test devices/VMs.
